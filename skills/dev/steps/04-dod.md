# Step 4: DoD

> 定义验收标准（Definition of Done）

---

## DoD 是什么

把 PRD 里的"成功标准"变成可勾选的清单：

```
PRD 成功标准: "用户能登录"
    ↓
DoD: - [ ] 用户输入正确密码能登录成功
     - [ ] 用户输入错误密码显示错误提示
```

---

## DoD 模板

```markdown
# DoD - <功能名>

## 验收标准

### 功能验收
- [ ] <功能点 1>
- [ ] <功能点 2>

### 测试验收
- [ ] npm run qa 通过
```

---

## 示例

```markdown
# DoD - 用户登录功能

## 验收标准

### 功能验收
- [ ] 用户输入正确密码能登录成功
- [ ] 用户输入错误密码显示错误提示
- [ ] 空用户名不能提交

### 测试验收
- [ ] npm run qa 通过
- [ ] 登录函数有单元测试
```

---

## 完成后

```bash
echo "✅ Step 4 完成 (DoD 确认)，可以开始写代码"
```

**DoD 完成后，进入 Loop 1：本地 QA**
