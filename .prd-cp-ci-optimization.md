## PRD - CI 优化：删除 Nightly workflow 和提升性能

**需求来源**: CI 深度调查发现 Nightly workflow 持续失败且不再需要，同时发现 CI 配置可优化（job 依赖串行、代码冗余）

**功能描述**:
1. **删除 Nightly workflow** - `.github/workflows/nightly.yml` 文件已持续失败，决定暂时移除
2. **优化 CI job 依赖** - 将快速检查改为并行执行，减少总运行时间
3. **创建 Composite Action** - 提取重复的 setup-project 逻辑，减少代码冗余

**涉及文件**:
- `.github/workflows/nightly.yml` - 删除
- `.github/workflows/ci.yml` - 优化 job 依赖关系
- `.github/actions/setup-project/action.yml` - 新建 Composite Action
- `regression-contract.yaml` - 移除 Nightly 相关的 trigger 配置

**成功标准**:
1. nightly.yml 文件已删除
2. ci.yml 的快速检查（version, known-failures, config, impact, contract-drift）改为并行执行
3. 创建了 setup-project composite action，ci.yml 中使用该 action
4. regression-contract.yaml 中移除了 Nightly trigger
5. CI 运行时间减少约 50-60 秒（baseline: PR workflow ~74秒，优化后预期 ~10-20秒，通过并行化快速检查）
6. npm run qa 通过
7. CI 通过，无新增失败

**非目标**:
- 不修改测试逻辑本身
- 不改变分支保护规则
- 不影响现有的 PR/Release 检查流程
- 暂不实现 Reusable Workflows（留待后续优化）
