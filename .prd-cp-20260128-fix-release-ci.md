# PRD: Release PR 跳过 DevGate 检查

## 背景

Release PR (develop → main) 不需要 DoD 文件，但 DevGate checks 会检查 DoD 存在性并失败。

## 功能描述

添加 `release:` 前缀的 PR 到 DevGate checks 跳过列表，使 release PR 可以正常通过 CI。

## 成功标准

1. release: 开头的 PR 跳过 DevGate checks
2. CI 通过

## 范围

- 修改 `.github/workflows/ci.yml` DevGate checks 条件
