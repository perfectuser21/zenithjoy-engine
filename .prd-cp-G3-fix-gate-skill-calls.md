## PRD - 修复 dev 工作流 subagent 调用方式

**需求来源**: 用户反馈 - 当前所有 gates 都用 Task(general-purpose) 调用，需要改为 Skill() 调用

**功能描述**:
将 dev 工作流中 6 个 gates 的调用方式从 Task(general-purpose) 改为 Skill() 工具调用：
- gate:prd (Step 1)
- gate:dod (Step 4)
- gate:qa (Step 4)
- gate:audit (Step 5)
- gate:test (Step 6)
- gate:learning (Step 10)

**涉及文件**:
- `skills/dev/steps/01-prd.md` - 修改 gate:prd 调用方式
- `skills/dev/steps/04-dod.md` - 修改 gate:dod 和 gate:qa 调用方式
- `skills/dev/steps/05-code.md` - 修改 gate:audit 调用方式
- `skills/dev/steps/06-test.md` - 修改 gate:test 调用方式
- `skills/dev/steps/10-learning.md` - 修改 gate:learning 调用方式

**成功标准**:
1. 所有 5 个步骤文件中的 Task(general-purpose) 调用都改为 Skill() 调用
2. Skill 调用格式正确：`Skill({ skill: "gate:xxx" })`
3. 删除所有冗长的内联 prompt，改为简洁的 Skill 调用
4. 测试通过：CI 绿色
5. 文档版本号正确更新（每个文件的 frontmatter）

**非目标**:
- 不修改 gate skill 本身的实现（已完成）
- 不修改其他 steps 文件（如 02-detect.md, 03-branch.md 等）
- 不创建新的 gates 或 checklists（已完成）
