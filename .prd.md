# PRD - 完善 CI 规则：正确检测 PR 标题类型

## 问题描述

PR #300 修复的 CI 规则不work，因为：
- 使用了 `github.event.head_commit.message` 检测 commit 类型
- 在 PR context 中此字段无效/不可靠
- 导致 chore:/docs:/test: PRs 仍然需要PRD/DoD/Version更新

## 解决方案

修改 workflow 使用 `github.event.pull_request.title` 检测 PR 标题：
```yaml
if: |
  !startsWith(github.event.pull_request.title, 'chore:') &&
  !startsWith(github.event.pull_request.title, 'docs:') &&
  !startsWith(github.event.pull_request.title, 'test:')
```

## 影响范围

- `.github/workflows/ci.yml` Version Check
- `.github/workflows/ci.yml` L2A Check

## 优先级

P2 - 修复PR #300不完整的实现
