# PRD - 术语更新：Checkpoint → Task

## 需求来源

用户反馈：代码中的 "Checkpoint" 术语与官方 Claude Code Checkpoint（自动撤销功能）概念混淆。需要明确区分：
- **官方 Checkpoint**：自动撤销机制（Esc+Esc rewind）
- **我们的 Task**：开发单元（1 个 PR）

## 功能描述

### 1. 术语更新

**文档文件**：
- `skills/dev/steps/03-branch.md`：Checkpoint 命名规则 → Task 命名规则
- `docs/INTERFACE-SPEC.md`：CP-001 → T-001
- `docs/AUDIT-REPORT.md`：Checkpoint 示例更新

**命名格式**：
- CP-{序号} → T-{序号}
- "Checkpoint" → "Task"

### 2. 添加概念说明

在相关文档中说明：
- 官方 Checkpoint = 自动撤销（文件级别，自动保存）
- 我们的 Task = PR 单元（功能级别，手动规划）

## 涉及文件

修改：
- skills/dev/steps/03-branch.md
- docs/INTERFACE-SPEC.md
- docs/AUDIT-REPORT.md
- 其他包含 CP-xxx 或 Checkpoint 引用的文件
