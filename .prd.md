# PRD - RADNA 体系落地 (v9.0.0)

Priority: P0

## 目标

建立统一的质量保证体系（RADNA），终结 Gate/Regression/QA/Audit 的混乱。

## 背景

问题：把 QA/Audit/Gate/Regression 混在一起，每次不知道该改哪个文件。
根因：对象不一样，但混在一个桶里。

## RADNA 体系（4 层）

| 层级 | 名称 | 作用 | 文件 |
|------|------|------|------|
| L0 | Rules | 宪法 | docs/policy/ASSURANCE-POLICY.md |
| L1 | Contracts | 契约（GCI + RCI） | contracts/*.yaml |
| L2 | Executors | 执行器 | scripts/run-*.sh |
| L3 | Evidence | 证据 | artifacts/*.md |

## 核心规则

- **GCI (Gate Contract)**: 保护检查系统不会放错行
- **RCI (Regression Contract)**: 保护业务功能不会回归

**铁律**: Gate 永远不进 RCI，业务永远不进 GCI。

## 交付物

1. skills/assurance/SKILL.md
2. docs/policy/ASSURANCE-POLICY.md
3. contracts/gate-contract.yaml（从 regression-contract.yaml 拆出）
4. contracts/regression-contract.yaml（只留业务）
5. scripts/run-gate-tests.sh
6. artifacts/ 目录结构

## 成功标准

- 4 层结构完整
- GCI/RCI 分离清晰
- npm run qa 通过
- 版本号统一为 v9.0.0
