# PRD - Skills 架构重构

## 需求来源

用户审计发现三个 Skills（/dev, /qa, /audit）之间存在：
- 措辞矛盾（"调用" vs "参考"）
- 职责边界模糊
- schema 不固定

## 功能描述

重构三个 Skills 的措辞和结构：
1. /dev 从"调用 /qa /audit"改成"QA Node / Audit Node（参考规范）"
2. /qa 强化输出 schema
3. /audit 强化输出 schema，删除"可选"改为"必须"

## 成功标准

- /dev 不再使用"调用"措辞
- /qa 有固定的输出 schema
- /audit 有固定的输出 schema
- /audit 明确是"必须"而非"可选"
